## Context

- 长难句页当前将句子拆成 token 用于点击查词、选词加入生词本（`SentenceLearningView.vue`）。
- 用户需要**本地**辅助：浅红底划成分、对选中词做**顶部批注**，且**不落库**。
- 约束：保持现有学习/打卡/接口不变；批注与高亮仅内存（或 session 级 store），`onUnmounted` / 路由离开清空。

## Goals / Non-Goals

**Goals:**

- 在句子文本上支持**多段浅红高亮**（划分成分），操作直观、可撤销或清除。
- 支持**一个或多个单词**的选中语义，与现有「单选生词」可并存或明确模式切换。
- 顶部**固定区域**展示**多条批注**（每条：关联的词或词组标识 + 用户文本），在本次页面停留期间保留。
- 离开长难句页后状态清空，无后端字段。

**Non-Goals:**

- 同步到其他设备、刷新后恢复、导出。
- 修改 `user_sentence_progress`、句子表或任何 RPC。
- 复杂富文本编辑器（Markdown 等）；纯文本即可。

## Decisions

1. **状态存放位置**  
   - **默认**：在 `SentenceLearningView.vue` 内用 `ref`/`reactive` 保存 `highlights[]`、`annotations[]`，`onBeforeUnmount` 清空。  
   - **备选**：若子组件拆分较多，可用 **Pinia 模块 `sentenceAnnotateSession`**，`storeId` 绑定路由或显式 `reset()`  on leave。  
   - **理由**：无持久化需求，优先组件内状态，减少全局污染。

2. **浅红高亮交互**  
   - **方案 A（推荐）**：在句子容器内监听 **pointer/mouse 按下—移动—抬起**，根据选区 `window.getSelection()` 或元素内字符偏移，将 `[start,end)` 存入 `highlights` 数组，渲染时用 **span 包裹**对应片段并加 `bg-red-100`（或设计 token）类。重叠高亮可合并为多条独立 span 或按时间顺序叠放（实现时选简单策略：允许多段不合并）。  
   - **方案 B**：进入「划成分」模式后每次拖拽增加一段。  
   - **未选定时**：与现有点击单词冲突——**决策**：增加轻量 **模式切换**（如「划成分 / 默认」toggle）或 **长按**进入划选，避免与单击查词冲突。  
   - **理由**：任意上浅红需明确与 click 查词的互斥或修饰键。

3. **多词选中与批注**  
   - 在**默认选词**（当前单选数组）基础上，扩展为 **批注专用多选**（例如 `Ctrl/Cmd` 多选 token，或框选连续词）。  
   - 用户点击「添加批注」或选中后自动在顶部追加一条 **卡片**：展示所选词拼接 + `textarea`/`input`，失焦或按钮确认后写入本地 `annotations[]`。  
   - **展示**：视口 **sticky top** 或句子**上方**固定条容器，多条批注纵向堆叠，可删除单条。

4. **与现有功能关系**  
   - 查词 `lookupWord`、生词本流程**保持不变**。  
   - 若高亮 span 包裹了单词，点击事件需 **stopPropagation** 或在高亮层不触发查词——**决策**：划成分模式下禁用查词点击，或仅未高亮区域可点词。

5. **无障碍与移动端**  
   - 触摸设备划选依赖浏览器 Selection API；若体验差，可文档化「桌面优先」或提供「添加当前选中为批注」按钮。

## Risks / Trade-offs

- **[Risk] 选区与 Vue 虚拟 DOM 冲突** → 用数据驱动渲染（句子拆成 fragment + v-for），避免直接 `innerHTML` 破坏绑定。  
- **[Risk] 高亮与 token 拆分不一致** → 以**字符偏移**相对整句 `content` 字符串为唯一真源，`splitSentence` 仅用于展示点击区。  
- **[Risk] 模式过多导致 UX 混乱** → MVP 先 **toggle：学习模式 / 划成分模式**，批注依赖明确的多选手势 + 顶部输入。

## Migration Plan

- 纯前端发布，无数据迁移。回滚：移除组件与状态即可。
