## Why

长难句训练中，学习者在口头拆解、圈画结构或随手标记时，仅靠「划成分 / 批注」有时不够自由；希望在**提交翻译之前**增加一层**临时涂鸦**，不打断现有句子交互流程，且数据可随页面离开丢弃，避免污染后端。

## What Changes

- 在句子学习页（长难句训练）增加**涂鸦模式**入口（例如「涂鸦」开关或按钮），与「划成分」「批注选词」并列或排在同一工具区。
- 进入涂鸦模式后，用户可在**限定区域**（至少覆盖英文原句卡片）内用手指/触控笔/鼠标**自由笔划**绘制。
- **开启涂鸦时强制关闭**「划成分」「批注选词」，避免模式冲突；关闭涂鸦后用户可再自行打开。
- 涂鸦层需为**会话内临时**：离开页面或组件销毁时清除；**翻译提交成功后必须清空**涂鸦；并提供「清除涂鸦」与**撤回上一笔**。
- 涂鸦激活期间，与句子的**点词、划成分、批注选词**的指针事件**互斥**（具体见 design）。
- **不落库**：涂鸦不写入 Supabase 或现有学习 API；若未来要导出，仅考虑前端截图级能力，本变更不要求。

## Capabilities

### New Capabilities

- `sentence-learning-doodle`: 长难句学习页在提交翻译前的本地会话涂鸦覆盖层、模式切换、与现有句子交互的共存规则及生命周期。

### Modified Capabilities

- （无）仓库 `openspec/specs/` 下暂无已建档主规格与本行为冲突；若后续合并主 spec，可将本能力并入「长难句学习」总规。

## Impact

- **前端**：`SentenceLearningView.vue`（或抽薄 composable + 子组件）增加 Canvas/SVG 覆盖层与指针绘制逻辑；可能与现有 `pointerdown/move/up` 句子交互需要统一分发。
- **API / 数据**：无新增远程接口。
- **依赖**：优先浏览器原生 Pointer Events；无需新 npm 依赖（除非设计选定第三方绘图库）。
