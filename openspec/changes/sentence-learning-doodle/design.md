## Context

- 长难句学习页（`SentenceLearningView.vue`）已有英文句子的复杂指针交互：划成分、批注选词、生词选择、查义等。
- 用户希望在**提交翻译之前**增加**自由涂鸦**，用于临时圈画，与结构化标注互补；数据保持**前端会话级**，不与后端同步。

## Goals / Non-Goals

**Goals:**

- 提供「涂鸦」模式开关；开启后用户可在**英文原句区域**（以原句卡片为边界）内自由绘制笔画。
- 涂鸦与句子底层文字**叠放清晰**：笔画在上层，不改变句子 DOM 与现有划成分/批注的渲染逻辑。
- 涂鸦模式开启时，**不触发**句子的点词、划选、批注选词等逻辑（避免与绘制竞争）；**同时强制关闭**「划成分」「批注选词」开关（产品已定）。
- 关闭涂鸦模式或离开页面后，行为回归现有交互；涂鸦数据不持久化到远程。

**Non-Goals:**

- 将涂鸦保存到数据库或多端同步。
- 笔刷商城、图层管理、复杂矢量编辑。
- 对整个页面（含翻译框、顶栏）全屏涂鸦（首版限定在原句卡片区域即可，可按产品后续扩展）。

## Decisions

1. **覆盖层技术：`<canvas>` + Pointer Events**  
   - **理由**：自由笔划连续路径在 Canvas 2D 上实现简单、移动端性能可接受；**笔划栈 + 重绘**即可实现「撤回上一笔」。  
   - **备选**：纯 SVG 每划一条 `<path>` — 撤销可删节点；首版以 Canvas 笔划栈为准。

2. **涂鸦区域锚定**  
   - 覆盖层**绝对定位**在「英文原句」外层容器上，`position: relative` 父级 + canvas `absolute inset-0`；Canvas 尺寸与容器同步（`ResizeObserver` 或 `onMounted` + `window.resize`，并处理 `devicePixelRatio` 缩放避免模糊）。  
   - **理由**：与用户截图一致，圈画针对句子；翻译输入区保持系统原生行为。

3. **与现有交互互斥 + 强制关模式**  
   - `doodleMode === true` 时：原句段落上的 `pointerdown/move/up` **不注册或由父级短路**（例如在覆盖层 `touch-action: none` 并 `pointer-events: auto`，句子容器 `pointer-events: none` 仅在该模式下）。  
   - **打开涂鸦的瞬间**：将 `constituentMode`、`notePickMode`（及与批注选词相关的临时态，如 `notePickSpans`、未完成弹窗）**一律关掉**，与现有 `watch`/清理逻辑一致，避免用户以为仍可划成分/批注。  
   - **关闭涂鸦**后恢复 `pointer-events` 与既有 handler；用户需自行再开启划成分或批注。  
   - **理由**：比「仅禁用指针、开关仍亮」更不易误导。

4. **笔画数据与生命周期（支持撤回上一笔）**  
   - 内存中维护 **笔划列表**（每笔触数组：归一化或物理像素坐标点列 + 样式），展示时 **整画布重绘**（清空后按序 `stroke`），或维护离屏 bitmap + 笔划栈；**撤回上一笔** = 弹栈并重绘。  
   - **清除涂鸦** = 清空栈 + `clearRect`。  
   - **离开路由 / `onBeforeUnmount`**：清空栈与画布；不依赖远端。

5. **提交翻译时的处理**  
   - 涂鸦**不参与**提交 payload。  
   - **翻译提交成功后必须**清空涂鸦（清空笔划栈 + 画布），与单次任务边界一致；另保留「清除涂鸦」「撤回上一笔」供提交前使用。

6. **样式**  
   - 默认笔色：**深色可见**（如 `rgba(0,0,0,0.65)`）及 **线宽 ~2–3px**（按 DPR 缩放）；暗色主题可用略浅笔色（可用 CSS 变量或 `prefers-color-scheme` 在脚本里选色）。  
   - 无需颜色选择器 MVP；可作后续任务。

## Risks / Trade-offs

- **[Risk]** 高 DPR 设备上 Canvas 未缩放导致模糊 → **Mitigation**：按 `clientWidth/Height * dpr` 设置 `canvas.width/height`，CSS 尺寸保持逻辑像素。  
- **[Risk]** 划成分/批注的视觉层与涂鸦视觉混淆 → **Mitigation**：笔色与线宽与红/蓝体系区分（黑/灰实线）；涂鸦仅模式开启时出现。  
- **[Risk]** 小屏上开关过多 → **Mitigation**：涂鸦与现有模式同区展示；**开启涂鸦即关掉**划成分/批注，减少并存状态。

## Migration Plan

- 纯前端增量；无数据库迁移。发布后可随时回滚：移除按钮与覆盖层组件即可。

## Resolved product decisions

1. **开启涂鸦** → **强制关闭**「划成分」「批注选词」及相关临时选词/弹窗状态。  
2. **提交翻译成功** → **必须清空**涂鸦。  
3. **撤回上一笔** → **必须实现**（笔划栈 + 重绘或等价方案）。
